# DeepSeek Agent

CLI AI-агент на базе DeepSeek API.

## Быстрый старт

```bash
cp .env.example .env
# вставить DEEPSEEK_API_KEY в .env
npm start
```

## Как это работает

### Agent Loop (`src/agent.js`)

Главный цикл агента. Отправляет сообщения в DeepSeek, получает ответ со стримингом,
обрабатывает вызовы инструментов и повторяет до получения финального ответа.

```
Сообщение пользователя
        ↓
    Вызов LLM (стриминг)
        ↓
  ┌─ tool_calls? ──→ показать инструмент ──→ запросить подтверждение (если опасный)
  │                       ↓
  │               выполнить инструмент
  │                       ↓
  │               добавить tool_result в messages ──────────────────────────────┐
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────── ↑
        ↓
      stop → вернуть ответ
```

Использует **стриминг** (`stream: true`) — ответ модели печатается в терминал по мере генерации.

### Инструменты (Tools)

Каждый инструмент имеет единую структуру:

```js
{
  name: "...",            // имя, по которому модель вызывает инструмент
  description: "...",    // описание для модели — когда и зачем использовать
  parameters: { ... },   // JSON Schema входных параметров
  isReadOnly: true/false, // определяет нужно ли подтверждение пользователя
  execute(args) { }      // реализация
}
```

#### Безопасные инструменты (без подтверждения)

| Инструмент | Файл | Описание |
|---|---|---|
| `read_file` | `tools/read.js` | Читает файл по пути, возвращает содержимое |
| `glob` | `tools/glob.js` | Находит файлы по паттерну (`src/**/*.js`) |
| `grep` | `tools/grep.js` | Ищет строку/regex в файлах, возвращает совпадения с номерами строк |

#### Опасные инструменты (требуют подтверждения `[y/N]`)

| Инструмент | Файл | Описание |
|---|---|---|
| `bash` | `tools/bash.js` | Выполняет shell-команду, возвращает stdout/stderr |
| `write_file` | `tools/write.js` | Создаёт или перезаписывает файл |
| `edit_file` | `tools/edit.js` | Заменяет точную строку в файле, показывает diff перед записью |

#### Подробнее о `edit_file`

Модель передаёт `old_string` и `new_string`.
Инструмент проверяет что `old_string` встречается ровно один раз (иначе отказывает),
показывает diff и ждёт подтверждения.

```
- старая строка
+ новая строка
```

### UI (`src/ui.js`)

ANSI-цвета без внешних зависимостей:

| Цвет | Используется для |
|---|---|
| зелёный | префикс `Agent:` |
| голубой | название вызываемого инструмента |
| жёлтый | запрос подтверждения `[y/N]` |
| серый (dim) | аргументы инструмента, результат, подсказки |
| красный | ошибки |

## Архитектурные принципы

- **isReadOnly** — разделение инструментов на безопасные и опасные, подтверждение перед деструктивными операциями.
- **Tool loop** — агент не останавливается после первого ответа, продолжает вызывать инструменты пока не решит задачу.
- **Стриминг** — ответ печатается по мере генерации, не ждёт полного ответа.
- **Системный промпт** — инструктирует модель читать файлы перед редактированием, исследовать кодовую базу через grep/glob.
