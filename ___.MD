# Developer Notes — DeepSeek Agent

## Добавить новый инструмент

1. Создать файл `src/tools/mytool.js`:

```js
export const myTool = {
  name: "my_tool",               // имя которое видит модель
  description: "...",            // описание для модели — от него зависит когда LLM вызовет инструмент
  parameters: {
    type: "object",
    properties: {
      foo: { type: "string", description: "..." }
    },
    required: ["foo"]
  },
  isReadOnly: true,              // false = пройдёт через checkPermission (alwaysAllow / запрос [y/N])
  async execute({ foo }) {
    // реализация
    return "результат строкой"   // всегда возвращать строку
                                 // исключение: изображение — см. раздел "Чтение изображений"
  }
}
```

2. Импортировать и добавить в массив `TOOLS` в `src/agent.js` (внутри `initialize()`):

```js
import { myTool } from "./tools/mytool.js"
TOOLS = [..., myTool]
```

Больше ничего менять не нужно — JSON Schema, маршрутизация, хуки, разрешения и `disallowedTools` подхватываются автоматически.

## Структура agent.js

```
agentLoop(userMessage)
  │
  ├─ initialize() — один раз при первом вызове:
  │    ├─ loadMcpTools() — подключает MCP-серверы
  │    ├─ фильтрует disallowedTools из конфига
  │    └─ initTaskTool(agentLoop) — инжектирует себя в task tool
  │
  ├─ runHooks("UserPromptSubmit")
  ├─ loadMemory() → добавляет AGENT.md в system prompt
  ├─ compactIfNeeded() → сжимает контекст если > 80% лимита
  │
  └─ while(true)
       │
       ├─ client.chat.completions.create({ stream: true, model: getModel() })
       │    ├─ печатает reasoning_content если deepseek-reasoner
       │    ├─ emit("text", ...) в json-режиме
       │    └─ собирает fullContent и toolCalls из дельта-чанков
       │
       ├─ finish_reason === "stop"
       │    ├─ runHooks("Stop")
       │    └─ return fullContent
       │
       └─ finish_reason === "tool_calls"
            └─ для каждого вызова:
                 ├─ runHooks("PreToolUse") — может заблокировать (exit code != 0)
                 ├─ checkPermission() — alwaysAllow / neverAllow / [y/N]
                 ├─ tool.execute(args)
                 ├─ если результат — изображение → формирует image_url content block
                 ├─ emit("tool_call") / emit("tool_result") в json-режиме
                 ├─ runHooks("PostToolUse")
                 └─ добавить { role: "tool", ... } в messages
```

Стриминг собирается вручную — `tool_calls` приходят по частям (delta), склеиваются по `tc.index`.

## Формат сообщений (OpenAI API)

```js
// пользователь
{ role: "user", content: "..." }

// ответ модели с вызовом инструмента
{ role: "assistant", content: null, tool_calls: [
  { id: "call_xxx", type: "function", function: { name: "read_file", arguments: '{"path":"..."}' } }
]}

// результат инструмента — текст
{ role: "tool", tool_call_id: "call_xxx", content: "содержимое файла..." }

// результат инструмента — изображение
{ role: "tool", tool_call_id: "call_xxx", content: [
  { type: "image_url", image_url: { url: "data:image/png;base64,..." } }
]}
```

`tool_call_id` в ответе должен совпадать с `id` из вызова — иначе API вернёт ошибку.

## Конфигурация (.agent/settings.json)

```json
{
  "model": "deepseek-chat",
  "thinkingModel": "deepseek-reasoner",
  "contextLimit": 60000,
  "alwaysAllow": ["read_file", "glob", "grep", "todo_read"],
  "neverAllow": [],
  "disallowedTools": [],
  "dangerouslyDisableSandbox": false,
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"]
    }
  }
}
```

| Поле | Описание |
|---|---|
| `alwaysAllow` | инструменты без запроса подтверждения |
| `neverAllow` | инструменты которые всегда отклоняются (модель видит, но выполнение блокируется) |
| `disallowedTools` | инструменты которые не регистрируются — модель их не видит вообще |
| `dangerouslyDisableSandbox` | снять ограничения sandbox в bash (по умолчанию `false`) |
| `mcpServers` | MCP-серверы, подключаются при старте |

Разница `neverAllow` vs `disallowedTools`: первый блокирует в рантайме, второй не даёт модели знать об инструменте.

## Sandbox в bash.js

По умолчанию `dangerouslyDisableSandbox: false` — bash блокирует:

| Паттерн | Причина |
|---|---|
| `curl`, `wget`, `fetch` | сетевые утилиты |
| `rm -rf /` | деструктивное удаление |
| `dd ... of=/dev` | запись в устройства |
| `chmod 777` | опасные права |
| `sudo`, `su` | повышение привилегий |

Чтобы разрешить: `"dangerouslyDisableSandbox": true` в `.agent/settings.json`.

## Хуки (.agent/hooks.json)

Хуки — shell-команды, запускаемые на события агента. Получают JSON через stdin.

```json
{
  "PreToolUse":       [{ "command": "cat >> agent.log" }],
  "PostToolUse":      [{ "command": "cat >> agent.log" }],
  "UserPromptSubmit": [],
  "PreCompact":       [],
  "Stop":             []
}
```

| Хук | Когда | Блокирует? |
|---|---|---|
| `PreToolUse` | перед вызовом инструмента | да (exit code != 0) |
| `PostToolUse` | после выполнения | нет |
| `UserPromptSubmit` | при отправке сообщения | нет |
| `PreCompact` | перед сжатием контекста | нет |
| `Stop` | когда агент завершил ответ | нет |

Payload в stdin хука:
```json
{ "tool": "bash", "args": { "command": "ls" } }         // PreToolUse
{ "tool": "bash", "args": {}, "result": "..." }          // PostToolUse
{ "message": "текст сообщения" }                         // UserPromptSubmit
{ "tokenCount": 48000 }                                  // PreCompact
{ "response": "текст ответа агента" }                    // Stop
```

## Память (AGENT.md)

Файлы читаются при каждом вызове `agentLoop` и добавляются в system prompt.

| Файл | Назначение |
|---|---|
| `~/.agent/AGENT.md` | глобальный — настройки для всех проектов |
| `.agent/AGENT.md` | проектный — описание текущего проекта |
| `AGENT.md` | корневой — инструкции в корне репозитория |

> Не использовать `CLAUDE.md` и папку `.claude` — зарезервированы Claude Code.

## Extended Thinking (deepseek-reasoner)

Запуск: `npm start -- --think`

Переключает модель на `deepseek-reasoner`. Рассуждения (`reasoning_content`) печатаются в dim между `[thinking]` / `[/thinking]`.

Управляется через `src/thinking.js`:
- `enableThinking()` — включить
- `getModel()` — вернуть нужную модель
- `printReasoning(chunk)` — напечатать delta рассуждений

---

## Компактор контекста (src/compactor.js)

Срабатывает когда контекст превышает 80% от `contextLimit`.

1. Оценивает токены (~4 символа = 1 токен)
2. Запускает хук `PreCompact`
3. Отдельным LLM-вызовом суммаризирует историю
4. Заменяет историю одним сообщением-резюме (системный промпт сохраняется)

## MCP (src/mcp.js)

MCP-серверы подключаются при инициализации. Каждый инструмент сервера регистрируется с именем `mcp__<server>__<tool>`.

```json
"mcpServers": {
  "fs": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "C:/Users/h1/projects"]
  }
}
```

MCP-инструменты тоже фильтруются через `disallowedTools`.

## Task / Subagent (src/tools/task.js)

Три режима работы:

```json
{ "description": "напиши тесты" }
```
Синхронный — ждёт результата.

```json
{ "parallel": ["задача 1", "задача 2", "задача 3"] }
```
Параллельный — `Promise.all`, все идут одновременно.

```json
{ "description": "долгая задача", "background": true }
```
Фоновый — возвращает `task_id` сразу, результат получить через `task_result`.

Для избежания циклических импортов использует паттерн инициализации:
```js
initTaskTool(agentLoop)  // в agent.js, после объявления функции
```

---

## Todo (src/tools/todo.js)

Задачи хранятся в памяти сессии (при перезапуске сбрасываются).

Статусы: `pending` → `in_progress` → `done`

Зависимости — задача не может перейти в `in_progress` пока не завершены все `blockedBy`:
```json
{ "action": "add", "text": "деплой", "blockedBy": [1, 2] }
```

Вывод `todo_read`:
```
● #1 написать тесты
◑ #2 ревью кода
○ #3 деплой [BLOCKED] (needs: #1, #2)
```

---

## Чтение изображений (src/tools/read.js)

Поддерживаемые форматы: PNG, JPG, JPEG, GIF, WEBP.

Инструмент возвращает JSON с маркером `__type: "image"`. `agent.js` распознаёт его и формирует `image_url` content block вместо текстового. Модель получает изображение визуально.

Работает только если модель поддерживает мультимодальность (DeepSeek VL, GPT-4o и др.).

---

## WebSearch (src/tools/web_search.js)

Поиск через DuckDuckGo HTML API — без ключа, без регистрации.

Парсит HTML-ответ регулярным выражением, извлекает заголовок, URL и сниппет. Лимит по умолчанию — 5 результатов.

## JSON-режим вывода (src/output.js)

Запуск: `--output-format=json`

Каждое событие — отдельная строка JSON в stdout:

```json
{ "type": "text", "text": "фрагмент ответа" }
{ "type": "tool_call", "tool": "bash", "args": { "command": "ls" } }
{ "type": "tool_result", "tool": "bash", "result": "file1.js\nfile2.js" }
```

В json-режиме весь цветной вывод (`print()`) подавляется — только JSON.

Использование в CI/скриптах:
```bash
echo "что делает index.js?" | node index.js --output-format=json
node index.js --output-format=json "напиши hello world"
```

## Worktree (src/worktree.js)

Запуск: `npm start -- --worktree`

Создаёт `git worktree` в `.agent/worktrees/<branch>` и переключает `cwd` агента.
При выходе (exit / Ctrl+C / SIGTERM) ветка и worktree удаляются автоматически.

Требует git-репозиторий в текущей папке.

## Подключение другой модели

В `src/agent.js` поменять `baseURL`, в `.agent/settings.json` — `model`:

```js
// DeepSeek (текущий)
{ baseURL: "https://api.deepseek.com" }   model: "deepseek-chat"

// OpenAI
{ /* без baseURL */ }                      model: "gpt-4o"

// Ollama (локально, бесплатно)
{ baseURL: "http://localhost:11434/v1" }   model: "qwen2.5-coder"

// Groq (быстрый бесплатный tier)
{ baseURL: "https://api.groq.com/openai/v1" }  model: "llama-3.3-70b-versatile"
```

## Зависимости

Только то что реально используется:

| Пакет | Где |
|---|---|
| `openai` | `src/agent.js` — клиент DeepSeek API (OpenAI-совместимый) |
| `@modelcontextprotocol/sdk` | `src/mcp.js` — MCP клиент |
| `fast-glob` | `src/tools/glob.js`, `src/tools/grep.js` |
| `dotenv` | `index.js` — загрузка `.env` |

Намеренно не используются внешние пакеты для:
- UI в терминале — голые ANSI-коды (`src/ui.js`)
- валидации — JSON Schema вручную в каждом инструменте
- HTTP — встроенный `fetch` (Node >= 18)
- работы с файлами — встроенный `fs/promises`

## Требования к окружению

В `package.json` прописано:

```json
"engines": { "node": ">=18.0.0" }
```

Минимум Node 18 — нужен для:
- `fetch` (встроенный, используется в `web_search.js`)
- `fs/promises` (стабилен с Node 14, но 18 LTS надёжнее)
- ESM (`import/export`)

При `npm install` на более старой версии Node выведет предупреждение.

---

## Известные ограничения

- `edit_file` — упрощённый diff (построчный), не unified
- `grep` — читает через fs, медленно на больших репозиториях (заменить на `ripgrep-node`)
- Todo-список — только в памяти сессии, при перезапуске сбрасывается
- Worktree — требует git
- `web_search` — парсит HTML DuckDuckGo, может сломаться при изменении их вёрстки
- Изображения — работают только с мультимодальными моделями
